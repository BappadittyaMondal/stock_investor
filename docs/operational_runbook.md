# HFOS v5.0 — Operational Runbook

## Overview
This runbook details standard operating procedures for resolving automated alerts generated by HFOS v5.0.

## Incident Escalation Paths
1. **P1 (CRITICAL): Database Connection Failure**
   - **Symptom:** UI returns 500 or Health Service reports database latency = -1.
   - **Action:** Check `system_errors` table (if accessible). SSH into server. Check SQLite file permissions or Postgres connection limits. Restart `docker-compose`.
   
2. **P2 (HIGH): Data Anomaly Detected**
   - **Symptom:** Operations Center shows a PRICE_GAP anomaly.
   - **Action:** Validate the stock ticker in question. If a corporate action (split) occurred, manually trigger `CorporateActionsService.apply_split()` to recalculate historical OHLCV. 

3. **P2 (HIGH): AI Quota Exhausted**
   - **Symptom:** AlertRouter fires P2. Copilot UI displays "Monthly AI budget exhausted."
   - **Action:** The system automatically falls back to disabling the AI feature. To restore, increase the `AI_MONTHLY_LIMIT_INR` in `.env` or wait until the 1st of the next month.

4. **P3 (MEDIUM): Feed Staleness**
   - **Symptom:** NSE API data > 1 day old.
   - **Action:** System automatically uses cached data. Confidence scores are downgraded. Check upstream API status.

## Audit Logs
All immutable security actions (Login, Weight Promotion, Paper Trade Approval) are stored in `audit_log`. For compliance audits, query `SELECT * FROM audit_log ORDER BY created_at DESC`.
